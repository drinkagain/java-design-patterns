### 代理模式
#### 三个角色的定义：
- Subject 抽象主题角色

    抽象主题类可以是抽象类也可以是接口，是一个最普通的业务类型定义，无
    特殊要求。
- RealSubject 具体主题角色

    也叫做被委托角色、被代理角色。是业务逻辑的具体执行者。
- Proxy 代理主题角色

    也叫做委托类、代理类。它负责对真实角色的应用，把所有抽象主题类定义的
    方法限制委托给真实角色实现，并且在真实主题角色处理完毕前后做预处理和
    善后处理工作。
